HTTP 端点

方法 / 路径作用
POST /auth/login用户名密码换 token
POST /auth/logout失效当前 token
GET /programs列出可 spawn 的程序
GET /proc列出当前用户的进程（ps）
POST /procspawn 新进程
POST /proc/<pid>/attach关联沙箱（首次/唤醒/重连自动 dispatch）
POST /proc/<pid>/hibernate释放沙箱（休眠）
GET /proc/<pid>/fs?path=…只读读取进程目录文件

所有请求在 header 带 token；失败返回 401。

WebSocket 流

端点 /proc/<pid>/stream，行分隔 JSON。下行（引擎 → 前端）：

{"type":"agent.thinking",    "id":"e8",  "delta":"…"}
{"type":"agent.tool_use",    "id":"e9",  "tool":"Bash", "input":"…"}
{"type":"agent.tool_result", "id":"e9",  "ok":true, "output":"…"}
{"type":"agent.message",     "id":"e10", "delta":"…"}

上行（前端 → 引擎）：

{"type":"user.input", "text":"…"}

聚合规则（按 id 追加、tool_use↔tool_result 配对）见 事件流协议 · 恢复与压实。

会话记录 schema

持久化在进程目录 session.aprog.jsonl 的合并后事件（非 wire delta）。每行一条：

{"id":"e8",  "ts":"…", "turn":3, "kind":"thinking", "text":"…", "dur":"2.4s"}
{"id":"e9",  "ts":"…", "turn":3, "kind":"tool", "name":"bash", "arg":"rg \"foo\"", "result":"…"}
{"id":"e10", "ts":"…", "turn":3, "kind":"reply", "text":"…"}
{"id":"u11", "ts":"…", "turn":4, "kind":"user", "text":"…"}

它是 engine-neutral 的逐字记录，给前端重放与审计用，不参与 resume。详见 事件流协议。

DB schema 概要

Control Plane 持久化的实体（字段详见 数据模型）：

users         (id, username, password_hash, created_at, last_login_at)
auth_sessions (id/token, user_id, created_at, last_used_at, expires_at, revoked_at)
processes     (pid, program, user_id, state, phase, checkpoint_ref,
               sandbox_id, sandbox_started_at, created_at,
               last_attached_at, last_active_at)

state 仅 spawned / running / hibernating（无终止态：跑完 / kill 都回到 hibernating，状态永不删）。 checkpoint_ref 是进程 git 仓库 HEAD 的 commit SHA；进程状态以 git 仓库存储（git push / clone 到 CP 托管 bare repo），无文件系统快照路径。详见 数据模型 · process 与 进程状态存储（Git 模型）。 Program 不入库——由文件系统扫描得到。

CLI（附录，已弱化）

已弱化 产品当前以 Web 前端为主入口。CLI 作为同协议的另一种客户端保留，命令表面如下，便于脚本化与调试。
命令作用
aprog login / logout登录 / 登出
aprog programs列出程序
aprog ps列出进程
aprog spawn <program>新建进程
aprog attach <pid>连上进程（TUI）；smart attach 自动 dispatch 首次/唤醒/重连
TUI 内 /detach / /exit离开 / 归档
