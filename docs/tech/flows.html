Process 生命周期

主线: spawn → attach mode 1 → (/detach → attach mode 3)* → /exit → (attach mode 2 → ...)*

spawn

注册 process + 初始化快照骨架. 不起沙箱, 不烧 compute.

attach · mode 1 — 首次

起沙箱 + 注入快照 + 启 driver + 打通流. spawn 后第一次接入.

attach · mode 2 — exit 后

跟 mode 1 几乎一样, 但快照含 session.jsonl, driver 起来时 SDK resume 对话续上.

attach · mode 3 — re-attach

沙箱在, driver 在. ring buffer 补 detach 期间事件, 然后续 live.

/detach

最轻量. CP 关 WS, 不通知 driver. 沙箱继续吃 compute.

/exit

tar-out 沙箱 → 覆盖快照 → 销毁沙箱 → state=exited.

认证 / 鉴权

login

argon2id 校验 + AuthSession 入库 + CLI 写 credentials.

logout

服务端 revoke + 客户端清 credentials. 总是 200.

每次请求

所有 endpoint 走的 token middleware. 滑动到期更新 last_used_at.
