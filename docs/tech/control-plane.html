设计要点

  • 无状态 service: CP 自己不缓存什么. 所有持久状态在 SQLite + 快照磁盘. 重启即可恢复任何在跑的 attach (沙箱还在, 重新 stream proxy).
  • 单一鉴权面: 所有 endpoint 走同一个 token middleware (除 /auth/login). 见 ADR-1.
  • Daytona adapter 是抽象边界: 换 sandbox provider 只动这一处 (sandbox.go). 见 ADR-3.
  • 流不入 DB: WS 事件实时透传, CP 不存. 历史靠快照里的 input.md + session.jsonl (那是 driver 自己写的). 见 ADR-4.
