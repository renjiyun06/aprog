设计要点

  • No ENTRYPOINT: 启动 driver 由 CP 显式 executeSessionCommand 控制. ENTRYPOINT 会跟 Daytona 的 session 概念打架.
  • esbuild 单文件 JS: 避免镜像里塞整套 node_modules; 启动也更快, 不用先 npm install.
  • 镜像只有一种 platform (amd64): Daytona 不支持 arm64 沙箱 (May 2026 现状).
  • 沙箱资源: snapshot push 时硬编码 2 vCPU / 4 GiB / 8 GiB disk. 后续可按 program 类型分档.
