HTTP REST API

所有 endpoint. JSON in, JSON out. Authorization: Bearer <token> 除了 /auth/login 都要带.

POST /auth/login

Request:
  Content-Type: application/json
  { "username": "lamarck", "password": "..." }

Response 200:
  { "token": "BWnX...32B base64url...",
    "user_id": "usr_abc",
    "username": "lamarck",
    "expires_at": 1751199600 }

Response 401:
  { "error": "invalid_credentials" }

Response 429:
  { "error": "rate_limited", "retry_after": 60 }

限流: 同 IP 5/min. 失败 5 次锁 username 10 分钟 (in-memory, MVP 不入库).

POST /auth/logout

Headers: Authorization: Bearer <token>

Response 200: { "ok": true }

幂等: token 已失效也返回 200, 不报错.

GET /proc

Query:
  ?all=1  → 含 exited (默认只列 spawned + running)

Response 200:
  { "items": [
      { "pid": "260521-x9k4",
        "program": "grasp",
        "state": "running",
        "phase": "concepts",
        "created_at": 1748549000,
        "uptime_seconds": 14432,
        "idle_seconds": 42,
        "last_attached_at": 1748563000,
        "last_exited_at": null,
        "sandbox_id": "sb_xxx" },
      ...
  ]}

列表只显示当前 user 的 process (按 user_id 过滤).

POST /proc

Request:
  { "program": "grasp" }

Response 200:
  { "pid": "260521-x9k4",
    "program": "grasp",
    "state": "spawned",
    "created_at": 1748549000,
    "snapshot_path": "/var/lib/aprog/proc/260521-x9k4/" }

Response 404: { "error": "program_not_found" }
Response 429: { "error": "quota_exceeded", "limit": 50 }

注意 spawn 不起沙箱, 不烧 compute. 见 CLI spawn / 快照生命周期.

POST /proc/{pid}/attach

Response 200:
  { "mode": 1,                  // 1 / 2 / 3, 由 CP 自动判定
    "pid": "260521-x9k4",
    "ws_url": "wss://api.aprog.dev/proc/260521-x9k4/stream?ticket=tk_abc...",
    "ticket_expires_at": 1748563205 }  // 5s 后过期

Response 404: { "error": "process_not_found" }
Response 503: { "error": "sandbox_unavailable", "detail": "..." }

mode 选择: CP 根据 process 当前状态自动 dispatch:

  • sandbox_id NULL + state=spawned → mode 1
  • sandbox_id NULL + state=exited → mode 2
  • sandbox_id != NULL → mode 3

客户端不需要选 mode. response 给 mode 主要让 TUI 展示一个 hint ("starting fresh sandbox..." vs "resuming..." vs "reconnecting...").

错误一致性

所有错误统一 shape:

{ "error": "<machine-readable kebab-case>",
  "message": "<optional human-readable>",
  "detail": "<optional extra fields>" }

常见错误码

code含义客户端行为
unauthorizedtoken 缺失 / 失效 / revoked清 credentials, 提示 login
invalid_credentialslogin 时 username/password 错提示重输
program_not_foundprogram name 不存在于文件系统报错退出
process_not_foundpid 不存在或不属于当前 user报错退出
sandbox_unavailableDaytona 起沙箱失败提示稍后重试
quota_exceededper-user 配额超限提示并打印 limit
rate_limited请求过频按 retry_after sleep

curl 速查

# login
curl -X POST https://api.aprog.dev/auth/login \
  -H 'content-type: application/json' \
  -d '{"username":"lamarck","password":"..."}'

# 后续都带 token
TOKEN=...
curl -H "authorization: bearer $TOKEN" https://api.aprog.dev/programs
curl -H "authorization: bearer $TOKEN" https://api.aprog.dev/proc

# spawn
curl -X POST https://api.aprog.dev/proc \
  -H "authorization: bearer $TOKEN" \
  -H 'content-type: application/json' \
  -d '{"program":"grasp"}'

# attach (拿 ws_url 后用 websocat 之类连)
curl -X POST https://api.aprog.dev/proc/260521-x9k4/attach \
  -H "authorization: bearer $TOKEN"
