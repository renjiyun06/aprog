设计要点

  • 没有 socket: 之前的设计文档曾经写过"driver 开本地 socket", 后来想清楚: Daytona 的 session command 本身就是 stdio 管道, driver 用 stdin/stdout 就够了. 见 tech-design 修订.
  • engine 不知道 aprog: SDK 拿到的 prompt / hooks / options 都是 SDK 自己语义的. aprog 协议事件由 translate.ts 单点生成. 换 engine 只动 driver, 不动 SDK.
  • state-skill 协议走 hooks: UserPromptSubmit 自动把用户提示 append 到 input.md 并推进 input_cursor. program 自己不用关心这件事. 见 hooks 详细.
  • /exit 是 driver 视角下的 "generator return": 这是 streaming-input 模式的设计 —— prompt iterable 自然结束 → SDK 关 session → query() loop 退出 → driver process.exit(0).
