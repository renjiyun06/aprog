Documentation

aprog 是一台 "AI 程序运行的虚拟 OS" —— agent 是引擎, program 是软件, sandbox 是环境. 这里收集平台的命令表面 / 数据模型 / 架构形态三份核心规约, 按序读完可建立完整心智模型.

01 · CLI Reference

aprog 平台的命令行入口. login / logout / programs / ps / spawn / attach, 加上 TUI 内的 /detach 和 /exit. smart attach 根据沙箱是否存在自动 dispatch 三种内部 mode.

02 · Data Model

User / AuthSession / Program / Process 四个实体, 字段表 + 关系图 + 状态机. 快照模型: 沙箱在时工作副本权威, 沙箱不在时平台快照权威. 状态永不删除.

03 · Architecture

Program / Engine / Sandbox 三层抽象. 组件 / 通信链路 / 部署拓扑 / 代码分层, 以及 4 条 ADR 锁定 load-bearing 的设计决策 (代理 vs 直连, 沙箱内常驻 driver, engine 可替换, session.jsonl 入快照).

04 · Technical Design (map)

架构落地到具体技术栈, 分三层呈现. L1 地图: 视觉系统全景, 组件可点击. L2 组件聚焦: 一个组件一页, 内部模块图 + 子模块卡片. L3 参考: 完整 DDL / wire protocol / 时序图 / Dockerfile / 代码骨架. Go (CLI + control plane) + TypeScript (claude-driver) + Daytona.
