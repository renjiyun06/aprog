生命周期流程

把组件串起来看一次进程的生老病死：spawn → attach → detach → hibernate → wake → exit。 每一步谁调用谁、沙箱与快照如何变化。

spawn · 新建进程

用户在窗口里点 ＋ 填个名字。Control Plane 在 snapshot_path 写入骨架并不起沙箱：

前端POST /proc {program, name}
↓
🧭 Control Plane
  • 验 token → User
  • 生成 pid（yyMMdd-nanoid）
  • 写快照骨架：meta.yml + 空 input.md + 空 execution-state/
  • DB 插入 Process(state=spawned, sandbox_id=null)
  • 返回 pid

此时进程休眠（没有沙箱）。状态：spawned。

attach · 关联沙箱并连上对话

第一次 attach（或从休眠 wake）会创建沙箱、注入快照、启动 Driver+Engine，再建立流：

前端POST /proc/<pid>/attach
↓
🧭 Control Plane · 起沙箱
  • 验 token + owner
  • sandbox_id 为 null → 起新 Daytona 沙箱（镜像 aprog/base）
  • 写 sandbox_id + sandbox_started_at（state→running）
  • tar 注入 snapshot_path → 沙箱 /home/aprog/
  • daytona exec：后台启动 claude-driver
  • 返回 WS endpoint
升级 WS · /proc/<pid>/stream ▼↓
🪢 建立流（Bridge 管道）
  • daytona exec：aprog-bridge --from-offset=0
  • bridge 连上 driver socket
  • 下行：bridge.stdout → parse → WS.send
  • 上行：WS.recv → bridge.stdin → driver → engine

之后用户每输入一句，走 WS → Bridge → Driver socket → 喂给 query()，引擎跑 agent loop， 消息出 iterator → Driver 翻成 aprog 事件 → 写会话记录 + 回流前端渲染。详见 事件协议与会话。

detach · 离开但保留沙箱

用户离开会话（关窗口/断网）。前端关 WS：

前端关闭 WS
↓
🧭 Control Plane 关 Daytona exec bridge 退出
  • driver 不动，继续接 engine 输出写本地缓冲
  • 更新 Process.last_attached_at

沙箱还在，引擎可能还在跑。重新连上就是 reattach：跳过沙箱创建，--from-offset 用上次游标补回间隙事件。

hibernate · 释放沙箱

用户在进程行点“休眠”按钮，显式释放沙箱：

前端POST /proc/<pid>/hibernate
↓
🧭 Control Plane 释放沙箱
  • tar-out 沙箱 /home/aprog → 覆盖 snapshot_path（回收最新状态）
  • destroy 沙箱
  • sandbox_id = null → 语义上的 休眠

进程目录与会话记录完整保留在平台快照里。状态：休眠。引擎私有的 session.jsonl 随沙箱一起消失——无所谓，它不权威。

wake · 重新关联沙箱

用户在休眠进程点“唤醒”按钮。等同于一次 attach：起新沙箱、注入快照、起 Driver。

冷启动靠状态 新沙箱里引擎是空上下文。它读进程目录的状态文件（meta.yml 的 phase/cursor、input.md、execution-state）重建工作记忆， 不重放历史对话。前端则照常铺出完整历史对话（读会话记录），对用户连续。原因见 唤醒靠状态，不靠对话。

exit · 归档

程序执行完毕，Driver 收到 SessionEnd：

DriverSessionEnd
↓
通知 🧭 Control Plane 归档
  • tar-out /home/aprog → 覆盖 snapshot_path
  • destroy 沙箱
  • sandbox_id = null, state = exited, last_exited_at = now

exited 不等于删除——只是没有沙箱关联。目录、产物、会话记录都还在，新输入到来还能重新 attach 回 running。 快照生命周期的事实源归属见 数据模型 · 快照。
