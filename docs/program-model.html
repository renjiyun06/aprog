程序与安装数据模型

登录后用户看到的是自己安装的智能程序。这页定义支撑它的数据模型： Program（智能程序 = skill，全局静态目录项）与 Installation（用户 × 程序的边，"装在我桌面上"）。 这套模型由前端实际设计反推而来，并刻意做到尽量简单。

范围 本页只覆盖 Program / Installation 两层。第三层 Process（程序的一次运行：spawn / ps / attach / hibernate / kill） 暂不建模，待运行时部分单独成文。系统应用（程序商店 / 设置）是前端 chrome， 不是智能程序，不进本模型。

三层与本页范围

跟程序有关的对象分三层，对应 OS 隐喻；本页只落定前两层。

层是什么持久在哪本页
Program智能程序 = skill，全局静态目录项磁盘 skill 注册表（权威）；DB 可选薄镜像✓
Installation用户 × 程序的边 —— "装在我桌面上"控制平面 SQLite✓
Process程序的一次运行（PCB + 进程目录）DB（PCB）+ ~/.aprog/<pid>/（权威态）暂缓

设计取舍（决定了模型形状）

这套模型是从前端的实际交互一步步收敛出来的，几条关键取舍把它压到了最简：

  • 安装即在桌面，没有"桌面 / 状态栏"摆放管理 → Installation 不含 surface / position 字段。
  • 状态栏（任务栏）= 当前已打开的程序，是运行时 UI 态（开窗即现、关窗即消），不落库，不存"固定/dock"关系。
  • 系统应用不是智能程序：程序商店、设置只是前端 chrome（商店为桌面常驻入口），不进 programs / installations。
  • 无程序详情页：程序只需目录级元数据（名称 / 版本 / 摘要 / 分类），更丰富的介绍若需要再读 SKILL.md，不在数据模型里冗余。
  • 卸载只是从桌面移除：进程与数据在别处（进程目录），重装即恢复 → Installation 行 硬删即可，无需软删/历史。

Program —— 程序目录

Program 就是一个智能程序（skill），平台范围内唯一。它的权威态是磁盘上的 skill 注册表 （SKILL.md frontmatter 提供名称、版本、摘要等），与"进程目录是权威态、文件不进库"的原则一致。 因此 installations.program_id 只引用 slug，不必 FK 到一张完整的 programs 表。

若后台需要纯 DB 查询 / 校验 / 加管控字段，可加一张薄镜像（skill 注册表的缓存，非权威）：

字段类型说明
idstringslug = skill id（稳定标识）
namestring展示名
versionstring来自 SKILL.md
summarystring一句话简介
categorystring商店分组（规划与设计 / 开发与质量 / 文档…）
iconstring图标引用（glyph 名 / 资源）
publisherstring发布者
取舍 派生为主：磁盘 skill 注册表权威，DB 仅 installations 引用 slug，programs 表按需做缓存镜像。 另一选项是把 programs 做成 DB 权威表（便于纯 DB 查询、加管控字段），代价是要和磁盘注册表做同步。待定（见文末）。

Installation —— 用户安装的程序

这是真正要新增的表：表达"用户 X 把程序 Y 装在了桌面上"。一个 (user, program) 至多一条。 沿用现有 db/index.ts 风格（TEXT 主键、ISO created_at 字符串）。 

CREATE TABLE installations (
  user_id      TEXT NOT NULL,   -- FK users.id
  program_id   TEXT NOT NULL,   -- 程序 slug（= skill id）
  PRIMARY KEY (user_id, program_id)   -- 兼作唯一性 + “是否已装”查询
);
CREATE INDEX installations_user ON installations(user_id);
字段类型说明
user_idstringFK users.id
program_idstring程序 slug；不强制 FK，因目录权威在磁盘
  • 装 = 插一行；卸 = 删该行（硬删）。重装即恢复——进程与数据不在此表。
  • 列出我的程序 = WHERE user_id = ?（桌面图标据此渲染）。
  • 是否已装 = 主键命中。

文件 vs DB 的分界

东西放哪为什么
程序目录（catalog）磁盘 skill 注册表（DB 可选缓存）静态、可派生，权威在 SKILL.md
谁装了什么DB installations需查询 / 并发写的关系数据
任务栏"已打开"不持久（运行时 UI 态）= 当前打开的 Process，随窗口生灭
进程运行态 / 产物DB（PCB）+ 进程目录属 Process 层，本页暂缓

待你拍板

  • Program 目录入库与否：推荐派生为主（磁盘权威 + 按需缓存镜像）；还是做成 DB 权威表？
  • 新用户默认装哪些：空桌面（仅前端"程序商店"入口），还是预装几个常用智能程序？

这两点定了，installations 表即可进 db/index.ts 迁移，并接上控制平面的 install / uninstall / list API。
